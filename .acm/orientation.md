## Active operational rules

- **Destination-level edits are read back to the operator before committing, not after.** (Two same-day corrections, 2026-07-04.)
- Before claiming the stranger-pickup bar is met, run a zero-context reader. Author self-verification only covers mechanism-preservation. (Demonstrated empirically, 2026-07-03.)
- When auditing quality or fidelity, compare against an external reference (source skills, spec, prior version) rather than self-reading alone — comparison-against-source has out-found self-review every time it's been tried.
- Before encoding an unconfirmed inference into a committed file, ask first if answerable in one turn.
- Don't assert cross-repo claims from memory — read the source repo's own trail first.
- Scores are plateau-relative diagnostics, disposable on destination-shift, never standing targets — and they must carry **both** halves of the destination (capability and cost) or they will silently re-weight the identity, as the voided baseline did.
